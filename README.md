#Titanic - Machine Learning from Disaster

## Опис проєкту
Цей проєкт виконано в рамках курсу машинне навчання. Головна мета — розв'язати задачу бінарної класифікації, застосувавши математичні методи аналізу даних та алгоритми машинного навчання для прогнозування ймовірності виживання пасажирів "Титаніка". 

Набір даних взято з класичного змагання на платформі Kaggle: Titanic - Machine Learning from Disaster (https://www.kaggle.com/competitions/titanic).

##  Математична постановка задачі
Маємо матрицю ознак $X$, де кожен рядок відповідає пасажиру, а стовпці — його характеристикам (вік, стать, клас квитка тощо). Необхідно знайти оптимальну функцію $f(X)$, яка з мінімальною похибкою наближає вектор цільових значень $y$ (де $0$ — пасажир не вижив, $1$ — пасажир вижив). 

##  Методика та підготовка даних
Для забезпечення стабільності алгоритму було проведено попередню підготовку граничних умов (даних):
* Кодування ознак: Текстові категоріальні змінні переведено в числовий формат (Жінки = 0, Чоловіки = 1).
* Обробка пропусків: Порожні значення у стовпцях віку та тарифу замінено на середні арифметичні значення по вибірці.
* Feature Engineering: Замість розрізнених даних про родичів на борту (SibSp та Parch), шляхом лінійної комбінації створено нову, більш репрезентативну ознаку — загальний розмір сім'ї (FamilySize).
* Зменшення розмірності: Для уникнення шуму відкинуто статистично незначущі або сильно зашумлені змінні (ім'я, номер каюти, номер квитка).

##  Алгоритм: Дерево рішень (Decision Tree)
У якості математичної моделі обрано DecisionTreeClassifier. 

Алгоритм працює як рекурсивний оптимізатор: на кожному кроці він розбиває багатовимірний простір ознак на підобласті, мінімізуючи цільову функцію — Критерій неоднорідності Джині (Gini Impurity):

$$I_G = 1 - (p_0^2 + p_1^2)$$

де $p_0$ та $p_1$ — ймовірності належності пасажирів до відповідного класу в конкретному вузлі.

Гіперпараметри моделі:
Було встановлено жорстке обмеження глибини дерева: max_depth=4. Це дозволило алгоритму розв'язати "нічиї" (спірні групи пасажирів, де початковий $I_G = 0.5$) завдяки додатковим крокам оптимізації, але при цьому уникнути перенавчання (overfitting) і зберегти здатність моделі до узагальнення.
